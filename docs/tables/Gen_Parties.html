    Table Gen_Parties

    Entity

    Entity: General.Contacts.Parties

    Contains base data for different kind of parties - companies, persons, departments, etc. Entity: Gen_Parties

    Summary

    Name Type Description
    Administrative_Region_Id uniqueidentifier The administrative region in which the party is situated.
    Area_Id uniqueidentifier The area in which the party is situated.
    Creation_Time datetime Readonly
    Creation_User nvarchar(64) Readonly
    Default_Product_Coding_System_Id uniqueidentifier When not null, specifies coding system for products, which is required by the party. The coding system is used primarily for document printouts and document import/exports.
    GLN nvarchar(13) Global Location Number used by EDI systems
    Id uniqueidentifier PK Unique party id
    Is_Active bit Specifies whether the current party is active in the system or not.
    Notes nvarchar(254)
    Parent_Party_Id uniqueidentifier Organizational unit (branch from the hierarchy of all parties) to which this party is referred to
    Party_Code nvarchar(16) Readonly The unique code of the party
    Party_Name nvarchar(254) ML, Readonly The name of the party
    Party_Type nvarchar(1) Allowed: C, L, P, S, V Type of party. Currently supported are P=Person, C=Company, S=Store, L=Company Location, V=Division
    Party_Unique_Number nvarchar(16) Readonly Unique number of the party (National number for persons, Registration number for companies)
    Row_Version timestamp
    Update_Time datetime Readonly
    Update_User nvarchar(64) Readonly
