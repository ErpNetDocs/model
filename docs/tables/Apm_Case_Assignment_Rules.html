Table Apm_Case_Assignment_Rules

Entity

Entity: Projects.Agile.CaseAssignmentRules

Contains rules for automatically assigning cases to users based on the values of the case fields, such as project, project area, case category, system state, user state, etc. Entity: Apm_Case_Assignment_Rules (Introduced in version 25.1.3.5)

Summary

Name Type Description
Assign_To_User_Id uniqueidentifier The user who will be automatically assigned to the case when the rule conditions are matched.
Assignment_Kind char(3) Allowed: ARS, PRS, COW, CUS, SUS Specifies the logic for determining who should be assigned to the case. If 'Specific User' is selected, the user specified in the 'Assign To User' field will be assigned. Otherwise, the assignee is determined based on predefined users responsible for the project, project area, case, and others.
Case_Assignment_Rule_Id uniqueidentifier PK
Case_Category_Id uniqueidentifier The case category to which the rule applies. NULL means the rule applies to all categories.
From_Date date Starting date of rule validity. NULL means no from date restriction.
Is_Active bit Indicates whether the current rule is active.
Notes nvarchar(max) Additional information or comments related to the rule.
Priority char(1) Allowed: 1, 2, 3, 4, 5 Priority when multiple rules match the criteria.
Project_Area_Id uniqueidentifier The project area to which the rule applies. NULL means the rule applies to all project areas.
Project_Id uniqueidentifier The project to which the rule applies. NULL means the rule applies to all projects.
Project_Milestone_Id uniqueidentifier The project milestone to which the rule applies. NULL means the rule applies to all milestones.
Row_Version timestamp
Rule_No int Consecutive number of the rule.
Stakeholder_Party_Id uniqueidentifier The stakeholder party to which the rule applies. NULL means the rule applies to all parties.
System_State char(1) Allowed: 1, 2, 3, 4, 5, 6, 7 The system state to which the rule applies. NULL means the rule applies to all user states.
To_Date date Ending date (inclusive) of rule validity. NULL means that the rule is valid forever.
User_State_Id uniqueidentifier The user state to which the rule applies. NULL means the rule applies to all user states.
