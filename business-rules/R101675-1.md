# R101675-1 Bundle - Clear Dependent Hierarchy
+----------------------+--------------------------------------------------------------------------------------------------+
| **Code**             | R101675-1                                                                                        |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Entity**           | @Crm.Pos.Bundles                                                                                 |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Name**             | ClearDependentHierarchy                                                                          |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Attribute**        | Multiple                                                                                         |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Layer**            | Front-End                                                                                        |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Events**           | AttributeChanged(EnterpriseCompany, PosLocation)                                                 |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Priority**         | Normal                                                                                           |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Modify**           | YES                                                                                              |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Applicable         | [ALL // no condition needed](xref:applicable-legislations)                                       |
| Legislations**       |                                                                                                  |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Action**           | Cascade reset downstream attributes to NULL.                                                     |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Description**      | Maintains the integrity of the POS bundle hierarchy by clearing child selections when a          |
|                      | parent property is modified. The cascade follows a strict top-down order:                        |
|                      |                                                                                                  |
|                      | 1. If EnterpriseCompany changes: PosLocationGroup, PosLocation and PosTerminal are set to NULL.  |
|                      | 2. If PosLocation changes: PosTerminal is set to NULL.                                           |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Message**          | N/A                                                                                              |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Version**          | Introduced: v.27.1                                                                               |
+----------------------+--------------------------------------------------------------------------------------------------+
| **Revocable**        | NO                                                                                               |
+----------------------+--------------------------------------------------------------------------------------------------+
