# R39667-2 SAF-T Declarations - Update Correction Start Date Registry On Release
| Name | Value |
| ---- | ----- |
| Code | R39667-2 |
| Entity | @Regulatory.Saft.SaftDeclarations |
| Name | UpdateCorrectionStartDateRegistryOnRelease |
| Attribute | CorrectionsStartDate |
| Layer | Back-End |
| Events | Release |
| Priority | Normal |
| Modify | YES |
| Applicable Legislations | ALL // no condition needed |
| Condition | If CorrectionsStartDate differs from registry key /saft/correction_start_date |
| Action | Set registry key /saft/correction_start_date = SaftDeclaration.CorrectionsStartDate |
| Description| When a SAF-T Declaration is released, if the document CorrectionsStartDate is different from the current value of the registry key /saft/correction_start_date, the system must update /saft/correction_start_date with the value from the released declaration. The updated registry value must be used as the default correction start date for future declarations. |
| Message | |
| Version | 2026.2 |
| Revocable | NO |
