# R39658 Accounting Voucher - Reporting Currency Amounts Equal Movement Amounts

| Name | Value |
| ---- | ----- |
| Code | R39658 |
| Entity | @Finance.Accounting.AccountingVouchers |
| Name | ReportingCurrencyAmountsEqualMovementAmounts |
| Attribute | AccountingVoucherLine.Debit, AccountingVoucherLine.DebitReporting, AccountingVoucherLine.Credit,AccountingVoucherLine.CreditReporting |
| Layer | Back-End |
| Events | Releasing + |
| Priority | Normal |
| Modify | NO |
| Action | If AccountingVoucherLine.Currency=EnterpriseCompany.ReportingCurrency<br> and DocumentDate <= EnterpriseCompany.ReportingCurrencyEndDate <br>OR EnterpriseCompany.ReportingCurrencyEndDate Is Null,<br> for each voucher line validate that Debit = DebitReporting and Credit = CreditReporting|
| Description| Upon releasing the voucher, if the document date is before the Reporting Currency End Date <br> and the currency of the voucher line is the enterprise company’s reporting currency, <br>each line is validated to ensure that Debit = Debit Reporting and Credit = Credit Reporting.|    
| Message | In line {LineNo} of document {DocumentText}, Debit amount is not equal to Debit Reporting amount<br> or Credit amount is not equal to Credit Reporting amount. <br>                                                        When the currency of the accounting voucher line is the reporting currency, <br>Debit must equal Debit Reporting and Credit must equal Credit Reporting.<br><br>_Sort Messages By Attribute: LineNo_ | 
| Introduced |v.26 SP9 |
| Revocable | YES |
