# R39819 WarehouseReconciliation - Check Valid Review Status

| Name | Value |
| ---- | ----- |
| Code | R39819 |
| Entity |Logistics.Wms.WarehouseReconciliations |
| Name | CheckValidReviewStatus |
| Attribute | WarehouseReconciliationDetails.ReviewStatus, WarehouseReconciliationDetails.CountedQuantity |
| Layer | Back-End |
| Events | ClientCommit |
| Modify | YES |
| Action | For all updated Warehouse Reconciliation Details:<br><br> If (ReviewStatus != ReviewStatus.GetDatabaseValue())<br><br>If (ReviewStatus == Created)<br>Validate that:<br>ReviewStatus.GetDatabaseValue() = Created<br><br>If (ReviewStatus == Started)<br>Validate that:<br>ReviewStatus.GetDatabaseValue() = Started<br><br>If (ReviewStatus == Finished)<br>Validate that:<br>ReviewStatus.GetDatabaseValue() = Finished<br><br>If (ReviewStatus == Recount)<br>No validation<br><br>If (ReviewStatus == Approved)<br>Validate that one of the following is true:<br>ReviewStatus.GetDatabaseValue() = Approved<br>ReviewStatus.GetDatabaseValue() = Finished<br>(ReviewStatus.GetDatabaseValue() = Recount) && (CountedQuantity != Null)<br>(ReviewStatus.GetDatabaseValue() = Cancelled) && (CountedQuantity != Null)<br><br>If (ReviewStatus == Cancelled)<br>Validate that:<br>ReviewStatus.GetDatabaseValue() != Started |
| Description | Description:<br><br>Validates the Review Status selected for all updated Warehouse Reconciliation Details on update at Client Commit.<br><br>The rule checks whether the newly selected Review Status is valid according to the current Review Status stored in the database for each detail line.<br><br>The validation is based on the current Review Status of each detail line.<br><br>Allowed transitions:<br>- Created -> only Created<br>- Started -> only Started<br>- Finished -> only Finished<br>- Recount -> always allowed<br>- Approved -> allowed from Approved, Finished, Recount with CountedQuantity, or Cancelled with CountedQuantity<br>- Cancelled -> allowed from any status except Started<br><br>If the selected Review Status is not valid for the current status of a detail line, the rule throws the corresponding error message for that line. |
| Message |The current Review Status for line no. "{LineNo}" is Created. You cannot change the Review Status to "{ReviewStatus}".<br><br>From the current status, the Review Status can only be changed to Recount or Cancelled.<br><br>The current Review Status for line no. "{LineNo}" is Started. This status means that there are active Warehouse Orders for counting this line, so the Review Status cannot be changed until the counting is finished.<br><br>From the current status, the Review Status can only be changed to Recount.<br><br><br><br>The current Review Status for line no. "{LineNo}" is Finished. You cannot change the Review Status to "{ReviewStatus}".<br><br>From the current status, the Review Status can only be changed to Recount, Approved, or Cancelled.<br><br><br><br>The current Review Status for line no. "{LineNo}" is Recount. You cannot change the Review Status to "{ReviewStatus}".<br><br>From the current status, the Review Status can only be changed to Approved or Cancelled. The Approved status can be set only if the Counted Quantity field has a value.<br><br><br><br>The current Review Status for line no. "{LineNo}" is Approved. You cannot change the Review Status to "{ReviewStatus}".<br><br>From the current status, the Review Status can only be changed to Recount or Cancelled.<br><br><br><br>The current Review Status for line no. "{LineNo}" is Cancelled. You cannot change the Review Status to "{ReviewStatus}".<br><br>From the current status, the Review Status can only be changed to Recount or Approved. The Approved status can be set only if the Counted Quantity field has a value. |
| Version | Introduced in: 26.2 |
| Revocable | NO |
