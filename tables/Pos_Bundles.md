# Table Pos_Bundles


## Entity

Entity: [Crm.Pos.Bundles](~/entities/Crm.Pos.Bundles.md)

Defines a POS bundle - a set of products treated as a single item at the point of sale. Entity: Pos_Bundles (Introduced in version 27.1.0.66)

## Summary

| Name | Type | Description |
| - | - | --- |
|[Activation_Date](#activation_date)|`datetime` |Date from which the bundle becomes active. Null indicates there is no activation date restriction.|
|[Deactivation_Date](#deactivation_date)|`datetime` |Date from which the bundle is no longer active. Null indicates there is no deactivation date restriction.|
|[Enterprise_Company_Id](#enterprise_company_id)|`uniqueidentifier` |The own company the bundle belongs to.|
|[Is_Active](#is_active)|`bit` |Indicates whether the bundle is active.|
|[Notes](#notes)|`nvarchar(max)` |Free-text notes about the bundle.|
|[Pos_Bundle_Code](#pos_bundle_code)|`nvarchar(16)` |Short code identifying the bundle.|
|[Pos_Bundle_Id](#pos_bundle_id)|`uniqueidentifier` `PK`||
|[Pos_Bundle_Name](#pos_bundle_name)|`nvarchar(256)` `ML`|Display name of the bundle.|
|[Pos_Location_Group_Id](#pos_location_group_id)|`uniqueidentifier` |The location group the bundle applies to. Null indicates it is not scoped to a location group.|
|[Pos_Location_Id](#pos_location_id)|`uniqueidentifier` |The location the bundle applies to. Null indicates it is not scoped to a single location.|
|[Pos_Terminal_Id](#pos_terminal_id)|`uniqueidentifier` |The terminal the bundle applies to. Null indicates it is not scoped to a single terminal.|
|[Row_Version](#row_version)|`timestamp` ||
